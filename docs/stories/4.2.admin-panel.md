# Story 4.2: Admin panel mínimo

## Status

Draft

## Story

**As a** Pedro (decisor),
**I want** uma página que lista os últimos 100 leads com filtro por tier,
**so that** posso auditar leads sem entrar no RD Station e validar que o pipeline funciona.

## Acceptance Criteria

1. Rota `/admin` protegida por basic auth com credenciais em env vars `ADMIN_USER` + `ADMIN_PASSWORD`
2. Lista os últimos 100 leads lidos de Vercel KV (onde são gravados em paralelo ao envio RD)
3. Cada linha: timestamp, nome, WhatsApp (mascarado: `(11) *****-1234`), tier, top 3 respostas
4. Filtro dropdown por tier (quente/morno/frio/todos) + busca por nome
5. Botão "Reenviar para RD" por lead (chama retry manual no API route)
6. Feature flag via env var `ADMIN_PANEL_ENABLED` — se `false`, rota retorna 404

## Tasks / Subtasks

- [ ] Middleware auth + feature flag (AC: 1, 6)
  - [ ] `src/middleware.ts` template completo em architecture §7.1
  - [ ] Valida `ADMIN_PANEL_ENABLED === 'true'` primeiro; se false → rewrite para `/404`
  - [ ] Basic auth parsing + comparação com `ADMIN_USER`/`ADMIN_PASSWORD`
  - [ ] Comparação constant-time para evitar timing attacks (`crypto.timingSafeEqual`)
- [ ] Admin API endpoints (AC: 2, 5)
  - [ ] `src/app/api/admin/leads/route.ts` GET: lista leads de KV (`KEYS lead:*`, MGET, sort by timestamp desc, limit 100)
  - [ ] `src/app/api/admin/resend/route.ts` POST: recebe `{ leadId }`, relê payload de KV, chama `sendConversion`, atualiza status
  - [ ] Ambos protegidos pelo middleware (path matcher inclui `/api/admin/:path*`)
- [ ] UI admin (AC: 3, 4)
  - [ ] `src/app/admin/page.tsx` — Server Component que fetch direto KV no render
  - [ ] Tabela com colunas: Timestamp, Nome, WhatsApp (mask), Tier, Top 3 Answers, Ações
  - [ ] Dropdown de filtro (server-side filter via query param `?tier=quente`)
  - [ ] Search input (client-side JS filter nos 100 resultados — suficiente)
  - [ ] Botão "Reenviar" dispara Server Action que chama POST /api/admin/resend
  - [ ] Feedback de loading + toast de sucesso/falha após retry
- [ ] Phone masking helper
  - [ ] `src/lib/utils/format.ts` função `maskPhone('+5511912345678')` → `'(11) *****-5678'`

**Dependency:** Stories 2.3 (leads gravados em KV), 2.4 (retry infrastructure).

**Critical rules:**
- Comparação de senha timingSafe obrigatória
- Admin é feature flag off-by-default em produção se necessário (rule 2 security)
- Middleware NÃO deve acessar secrets além do necessário

**Rate limit admin:** Não é crítico pois já tem basic auth — mas adicionar counter `rate:admin:{user}` 60/min é boa prática (opcional nesta story, tratar como follow-up).
