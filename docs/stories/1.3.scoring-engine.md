# Story 1.3: Scoring engine determinístico

## Status

Draft

## Story

**As a** Pedro (decisor),
**I want** poder ajustar pesos e regras de scoring editando um único JSON,
**so that** posso iterar o modelo sem redeploy de código.

## Acceptance Criteria

1. `config/scoring.json` define pesos por pergunta/resposta em 4 eixos (pet_ativo, gasto, dor, cobertura)
2. Função pura `calculateTier(answers, config)` retorna `{ tier: 'quente'|'morno'|'frio', score: number, breakdown: {...} }`
3. Thresholds de classificação configuráveis no JSON (ex.: `quente >= 70`, `morno 40–69`, `frio < 40`)
4. Testes unitários (Vitest) cobrem: respostas perfeitas → quente, médias → morno, mínimas → frio, eliminação por "sem pet"
5. `calculateTier` executa 100% client-side, sem fetch
6. Breakdown por eixo disponível no resultado final para debugging e para renderização do diagnóstico personalizado

## Tasks / Subtasks

- [ ] Criar scoring config (AC: 1, 3)
  - [ ] `config/scoring.json` conforme schema de architecture §11.2
  - [ ] `config/cep-coverage.json` com dataset placeholder + mecanismo de loader (pendente open question #5 — dataset real da Jofi)
- [ ] Implementar scoring engine (AC: 2, 5, 6)
  - [ ] `src/lib/quiz/scoring.ts` com função pura `calculateTier(answers, config)`
  - [ ] Suporta rules: `weights`, `numeric-range`, `cep-coverage`
  - [ ] Flag `eliminated: true` quando pet-ativo=nao (força tier=frio)
  - [ ] Retorna `breakdown: { pet_ativo, gasto, dor, cobertura }` por eixo
  - [ ] Zero imports de `next/server`, `window`, ou fetch — 100% puro
- [ ] Integrar com QuizContext (AC: 2)
  - [ ] Reducer `FINISH` chama `calculateTier` e grava tier no state
  - [ ] Breakdown fica acessível em `state.scoreBreakdown`
- [ ] Testes (AC: 4)
  - [ ] `src/lib/quiz/scoring.test.ts` com 6 test cases (exemplo em archi §9.1):
    - respostas perfeitas → quente, score ≥70
    - respostas médias → morno
    - respostas mínimas → frio
    - `pet-ativo=nao` → eliminated:true, tier=frio (sem importar outras respostas)
    - `plano-atual=sim-jofi` → penalty -50 (morno vira frio)
    - determinismo: 2 chamadas com mesmo input → output idêntico
